# 탭별 버퍼 독립성 요구사항 정의서

## 1. 문제 정의

### 현재 상황 (❌ 잘못된 구현)
- **증상**: 탭1에서 파일 A,B 열고, 탭2에서 파일 C,D 열었을 때, 탭1에서 `:bnext` (,l) 누르면 탭2의 파일 C,D로 이동됨
- **원인**: Vim의 기본 버퍼 시스템은 전역으로 동작하며, 모든 탭이 동일한 버퍼 풀을 공유
- **영향**: 탭을 사용한 작업 공간 분리가 무의미해짐

### 기대 동작 (✅ 올바른 구현)
- **탭1**: 파일 A,B만 순회 (,h ,l로 A↔B만 이동)
- **탭2**: 파일 C,D만 순회 (,h ,l로 C↔D만 이동)
- **원칙**: 각 탭은 완전히 독립된 버퍼 세트를 가짐

## 2. 핵심 요구사항

### MUST HAVE (필수)
1. **탭별 버퍼 격리**: 각 탭에서 열린 버퍼는 해당 탭에서만 접근 가능
2. **커스텀 네비게이션**: `,h` `,l`이 현재 탭의 버퍼만 순회
3. **Winbar 일치성**: Winbar에 표시된 버퍼와 실제 네비게이션 대상이 100% 일치
4. **탭 전환 시 컨텍스트 유지**: 탭 전환 후 돌아와도 버퍼 위치 유지

### SHOULD HAVE (권장)
1. **버퍼 이동 기능**: 특정 버퍼를 다른 탭으로 이동
2. **공유 버퍼 옵션**: 필요시 특정 버퍼를 여러 탭에서 공유
3. **버퍼 히스토리**: 탭별 최근 사용 버퍼 추적

### NICE TO HAVE (선택)
1. **세션 저장/복원**: 탭별 버퍼 구성을 세션에 저장
2. **버퍼 그룹핑**: 관련 버퍼를 그룹으로 관리

## 3. 기술적 구현 방안

### 방안 A: 탭별 버퍼 리스트 관리
```vim
" 각 탭페이지에 버퍼 리스트 저장
t:tab_buffers = [1, 5, 8]  " 버퍼 번호 리스트
```

### 방안 B: 버퍼 메타데이터 활용
```vim
" 각 버퍼에 탭 정보 저장
b:buffer_tabs = [1, 3]  " 이 버퍼가 속한 탭 번호들
```

### 방안 C: 커스텀 네비게이션 함수
```vim
" :bnext/:bprevious 대체
function! TabLocalBnext()
  " 현재 탭의 버퍼만 순회
endfunction
```

## 4. 검증 시나리오

### 시나리오 1: 기본 격리 테스트
1. 탭1에서 파일 A, B 열기
2. 탭2로 이동 후 파일 C, D 열기
3. 탭1로 돌아와서 `,l` 반복 → A,B만 순회 확인
4. 탭2로 이동해서 `,l` 반복 → C,D만 순회 확인

### 시나리오 2: 버퍼 추가/삭제 테스트
1. 탭1에서 새 파일 E 열기
2. `,l` 반복 → A,B,E 순회 확인
3. 탭2는 여전히 C,D만 순회 확인

### 시나리오 3: Winbar 일치성 테스트
1. Winbar에 표시된 버퍼 목록 확인
2. `,1` `,2` `,3` 숫자 키로 이동
3. `,h` `,l`로 순회
4. 모든 경우 Winbar 표시와 실제 동작 일치 확인

## 5. 성공 기준

✅ **완료 조건**
- [ ] 탭1의 `,h` `,l`이 탭1 버퍼만 순회
- [ ] 탭2의 `,h` `,l`이 탭2 버퍼만 순회
- [ ] Winbar 표시와 네비게이션 100% 일치
- [ ] 탭 전환 후에도 버퍼 위치 유지
- [ ] 새 버퍼 추가 시 현재 탭에만 추가
- [ ] 버퍼 삭제 시 현재 탭에서만 제거
