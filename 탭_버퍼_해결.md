# 탭/버퍼 표시 문제 해결

## 문제
- bufferline.nvim이 모든 전역 버퍼를 표시
- 실제 Vim 탭과 혼동 발생
- 탭 전환 시 버퍼가 섞이는 현상

## 원인
- bufferline은 **버퍼를 탭처럼 보이게** 하는 플러그인
- 실제 Vim 탭 페이지와는 다른 개념
- 전역 버퍼 리스트를 상단에 표시하여 혼란 야기

## 해결책
### 1. bufferline.nvim → tabby.nvim 교체
- bufferline 제거 (전역 버퍼 표시)
- tabby.nvim 설치 (실제 탭 페이지 표시)
- 각 탭마다 독립된 작업 공간 유지

### 2. 개선된 키 매핑
#### 탭 네비게이션 (기존 유지)
- `tn` - 새 탭 생성
- `td` - 탭 닫기
- `th` - 이전 탭
- `tl` - 다음 탭
- `tj` - 첫 탭
- `tk` - 마지막 탭

#### 버퍼 네비게이션 (새로 추가)
- `,h` - 이전 버퍼 (현재 탭 내)
- `,l` - 다음 버퍼 (현재 탭 내)
- `,bd` - 버퍼 삭제
- `,bb` - 버퍼 목록

#### 탭 관리 명령어 (새로 추가)
- `:TabInfo` - 현재 탭 정보 및 버퍼 목록
- `:TabBuffers` - 현재 탭의 버퍼 목록만 표시
- `:MoveBufferToTab <n>` - 버퍼를 다른 탭으로 이동

## 사용 시나리오

### 시나리오 1: 프로젝트별 탭 관리
```
탭1: 프론트엔드 작업 (React 컴포넌트들)
탭2: 백엔드 API (Node.js 파일들)  
탭3: 설정 파일 (package.json, tsconfig 등)
```

### 시나리오 2: 작업 유형별 탭
```
탭1: 코드 작성
탭2: 테스트 파일
탭3: 문서화
```

## 장점
✅ 각 탭마다 독립적인 버퍼 세트
✅ 탭 전환 시 컨텍스트 유지
✅ 실제 Vim 탭 페이지 사용 (혼동 없음)
✅ 직관적인 워크플로우

## 테스트 방법
1. Neovim 재시작
2. `tn`으로 새 탭 생성
3. 각 탭에서 다른 파일 열기
4. `th`/`tl`로 탭 전환 확인
5. `,h`/`,l`로 버퍼 전환 확인
