# Neovim 포맷팅 및 권한 문제 해결 요약

## 해결된 문제들

### 1. SQL 포맷팅 문제 (space+f 작동 안함)
- **원인**: sqlfmt 설정 파일 부재 및 conform.nvim 에러 처리 미흡
- **해결**:
  - `.sqlfmt.toml` 설정 파일 생성 (Snowflake dialect 지원)
  - `.sqlfluff` 설정 파일 생성 (dbt 템플릿 지원)
  - conform.nvim에 에러 알림 추가
  - sqlfluff 린터 임시 비활성화 (dbt 템플릿 파싱 문제)

### 2. 권한 우회 경고
- **원인**: Mason 도구의 Python 가상환경 실행 권한 문제
- **해결**: 
  - `fix_permissions.sh` 스크립트로 권한 자동 수정
  - Mason bin 디렉토리 실행 권한 설정
  - Python venv 실행 권한 복구

### 3. 자동 업데이트 실패 (Claude CLI)
- **원인**: npm global 디렉토리(/usr/local)가 root 소유
- **해결 방법**:
  ```bash
  # 옵션 1: sudo로 업데이트
  sudo npm update -g @anthropic-ai/claude-code
  
  # 옵션 2: npm prefix 변경 (권한 문제 없는 디렉토리)
  npm config set prefix ~/.npm-global
  echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
  source ~/.zshrc
  npm install -g @anthropic-ai/claude-code
  ```

## 새로 추가된 기능

### 디버그 명령어
- `:FormatDebug` - 포맷터 상태 확인
- `:FormatTest` - 포맷 테스트 실행
- `:ConformDebug` - Conform 상세 정보

### 설정 파일
- `.sqlfmt.toml` - sqlfmt 설정 (Snowflake, 120자 라인, 대문자 키워드)
- `.sqlfluff` - sqlfluff 설정 (dbt 템플릿 지원)
- `lua/formatter_debug.lua` - 포맷터 디버그 도구

### 개선된 키 매핑
- `<space>f` - 포맷팅 실행 시 성공/실패 알림 표시

## 테스트 방법

1. Neovim 재시작
2. SQL 파일 열기
3. `<space>f`로 포맷팅 테스트
4. 문제 발생 시 `:FormatDebug` 실행
